# Tanhwe Guest House — Design Rules

## Non-Negotiable Rules

1. Do not use random gradients.
2. Do not use glassmorphism.
3. Do not create oversized hero sections that waste space.
4. Do not use giant border radii everywhere.
5. Do not use heavy shadows.
6. Do not add decorative blobs.
7. Do not stretch room images.
8. Do not hardcode content that should come from admin settings.
9. Do not make the public website look like a generic SaaS landing page.
10. Do not make the dashboard look like a toy template.

## Preferred References

Use inspiration from:

- Airbnb listing cards and search flow
- Breather room detail page structure
- Clean real estate listing grids
- Desert Tech dashboard practicality
- Linear/Vercel clarity
- Shopify Admin organization

## Brand Use

Primary orange and secondary blue must come from the Tanhwe poster.

Use warm cream and sand backgrounds to make the site feel hospitable.

## Public UI

Must be:

- Clean
- Image-first
- Booking-focused
- Mobile friendly
- Warm but professional

## Admin UI

Must be:

- Compact
- Practical
- Consistent
- Table-friendly
- Role-aware

## Forms

Forms must be simple and grouped logically.

Use:

- clear labels
- helpful placeholders
- validation messages
- loading states
- success states
- error states

## Buttons

Primary button:

- Orange background
- White text
- Slight hover darkening

Secondary button:

- White/cream background
- Blue or dark text
- Border

WhatsApp button:

- Use WhatsApp action clearly
- Can use green only when necessary

Call button:

- Use phone icon
- Keep brand styling

## Mobile

Mobile must not feel like a compressed desktop.

Use:

- single-column layouts
- sticky booking CTA where helpful
- bottom WhatsApp/call actions where appropriate
- collapsible filters
