# Coder rules

1. Preserve providers and API contracts.
2. Use additive Drizzle migrations and never edit applied migrations.
3. Require roles on pages and mutations.
4. Validate untrusted input and log operational writes.
5. Keep database and provider clients lazy for build safety.
6. Preserve booking and document price snapshots.
7. Do not expose owner financials to staff.
8. Keep client components at interactive leaves.
9. Never commit secrets or production data.
10. Do not mark work complete with failing checks.
